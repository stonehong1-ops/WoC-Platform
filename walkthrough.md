# Hero 배너 이벤트 슬라이더 개편 완료 보고서

지적해주신 상단 Hero 배너의 단일 이벤트 구조를 최근 5개 슬라이더 구조로 전면 개편하고, 이벤트 팝업 및 상세 보기 관련 버튼 명칭을 '참여하기'로 변경했습니다.

---

## 1. 개선 내역

### 1) Hero 이벤트 배너 다중 슬라이더(최근 5개) 전환
- **스냅 스와이프 슬라이더 적용**:
  - 단일 이벤트 고정 노출 방식에서, 가로 스와이프가 지원되는 모바일 친화형 스냅 슬라이더 구조(`snap-x snap-mandatory flex overflow-x-auto`)로 전환했습니다.
  - 사용자는 좌우 스크롤을 통해 최근 등록된 최대 5개의 이벤트를 미끄러지듯 슥슥 확인하며 탐색할 수 있습니다.
- **슬라이드 전용 인디케이터 도트**:
  - 슬라이더 하단 중앙에 실시간 스크롤 오프셋을 트래킹하여 활성화된 슬라이드 위치를 흰색 도트로 정밀 표기하는 인디케이터 도트(`activeDotIndex`)를 부착했습니다.
- **안정적 데이터 쿼리**:
  - Firestore의 `events` 컬렉션에서 최근 시작 날짜(`startDate` desc) 기준 최대 30개를 조회하여, 현재 활성화된 소사이어티 컨텍스트(`societyId`)에 완전히 부합하는 최신 이벤트 5개를 추출해 슬라이더로 바인딩합니다.
  - 데이터가 아직 없는 경우를 대비한 유려한 폴백(Fallback) 기본 템플릿 카드 슬라이드도 구조적으로 온전히 유지했습니다.

### 2) 버튼 텍스트 변경
- **자세히 보기 -> 참여하기**:
  - 직관적인 행동 유도를 위해 Hero 내 상세 정보 팝업 진입 버튼 텍스트를 기존 `자세히 보기`에서 **`참여하기`**로 정밀 교체했습니다.

---

## 2. 자가 검증 리스트
- 로컬 개발 서버가 구동 중인 상태에서 홈 화면(`/home`)에 진입합니다.
- 상단 Hero 배너가 좌우 스냅 스크롤이 되는지 스와이프를 수행해 봅니다.
- 스와이프 위치에 따라 하단 중앙의 도트 인디케이터가 활성화 상태로 자연스럽게 동기화되는지 점검합니다.
- 이벤트 카드 내부의 주황빛 버튼 텍스트가 `참여하기`로 명확하게 변경되었으며, 이를 눌렀을 때 해당 이벤트의 세부 팝업 창이 정상으로 호출되는지 확인합니다.
