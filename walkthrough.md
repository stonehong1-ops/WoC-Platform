# 내 주변 라이브 배경색 명도 강화 및 실시간 데이터 바인딩 보고서

모바일 기기 액정에서 다소 흐릿하게 보이던 '내 주변 라이브' 카드의 배경색을 선명하게 강화하고, 포털 내 3대 핵심 구역(내 주변 라이브, 지금 라이브 추천, 최근 영상)에 실제 Firestore 라이브 포스트 데이터를 실시간으로 바인딩 처리했습니다.

---

## 1. 개선 내역

### 1) 모바일 가독성을 위한 내 주변 라이브 배경색 명도/채도 강화
- **배경색 딥퍼플 그라데이션 적용**:
  - 모바일(폰) 액정 환경에서 컬러가 연하게 흐려 보인다는 피드백에 따라 기존 `from-[#6A4CF6] to-[#987EFF]`에서 채도가 높고 더 깊은 딥퍼플 라벤더 컬러 조합인 **`from-[#4D24F3] to-[#805CFF]`**로 전면 강화했습니다.
  - 이로써 모바일 액정에서도 또렷하고 비비드한 라이브 홈 카드의 세련미가 확실하게 살아납니다.

### 2) Firestore 실시간 라이브 포스트 데이터 바인딩
- **실시간 데이터 구독 (`subscribeFeed`)**:
  - `galleryService.subscribeFeed` 채널을 열어 실제 사용자들이 등록한 포스트 데이터를 실시간으로 수집하고, 최신 시간순 정렬로 관리합니다.
- **내 주변 라이브 바인딩**:
  - 실제 포스트를 최신순으로 가져와 3개의 행에 썸네일, 실제 장소명(또는 작성자명), 시청자수 시뮬레이션, 거리를 매핑했습니다.
- **지금 라이브 추천 바인딩**:
  - 실제 최신 포스트 중 대표 1개(Main)와 서브 2개(Sub)를 썸네일 이미지 및 실제 캡션/작성자명으로 동적 구성했습니다.
- **순간을 느끼다 (최근 영상) 바인딩**:
  - 실제 비디오 피드 포스트들을 6개의 가로 스크롤 썸네일로 연결했습니다.
- **유연한 폴백(Fallback) 구조 보존**:
  - 데이터베이스가 비어있거나 불러오는 중일 때는 기획서에서 명시한 시안용 데이터가 기본으로 렌더링되도록 처리하여 첫 렌더링 시에도 UI 붕괴가 생기지 않도록 방어 로직을 이식했습니다.

---

## 2. 자가 검증 리스트
- 모바일 브라우저를 통해 라이브 탭(`/live`)에 진입합니다.
- '내 주변 라이브' 카드의 보라색 배경이 흐리지 않고 또렷하게 명암이 나타나는지 점검합니다.
- 리스트 내부의 항목들이 목데이터가 아닌 실제 피드 데이터(포스터 이미지, 업로더 이름, 실제 캡션 등)로 실시간 반영되어 뜨는지 확인합니다.
