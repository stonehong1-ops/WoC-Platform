# 5대 도메인 피플 표준 배지(UserBadge) 전격 확대 의사결정 노트 (Context Notes)

## 1. 5대 핵심 도메인 전수 이식 의사결정
- **의사결정**. 숙소(`StayDetail.tsx`), 분실물(`LostFoundDetail.tsx`), 중고장터(`ResaleItemDetail.tsx`), 그룹소개(`GroupAbout.tsx`), 게시판/댓글 상세(`PostDetailModal.tsx`) 등 5개 영역의 수동 아바타/이름 렌더링 코드를 전부 표준 피플 배지인 `UserBadge`로 일괄 대체합니다.
- **이유**. 각 도메인 화면들이 개별적인 하드코딩 방식이나 수동 텍스트 바인딩을 취하고 있어, 유저가 프로필 사진이나 한영 닉네임을 변경할 때 실시간으로 반영되지 못하고 화면마다 다르게 깨져 보이던 치명적인 정합성 결함을 해결하기 위함입니다. 공통 규격인 `UserBadge`로 통합할 시 Firestore 캐시 데이터와 반응형으로 페어링되어 플랫폼 전체의 인적 정보 무결성이 정점으로 완비됩니다.

## 2. 디자인 정밀성 보존 (Zero Design Deviation)
- **의사결정**. 각 화면마다 아바타의 스펙 크기가 `w-6`, `w-10`, `w-12`, `w-14` 등으로 다릅니다. 이들을 무리하게 한 크기로 일관화하여 레이아웃을 깨뜨리지 않고, 원래 존재하던 디자이너의 레이아웃 간격과 아바타 크기를 `UserBadge`에 `avatarSize` 속성으로 정밀 전달하여 0px 편차로 계승 적용합니다.
- **이유**. 플랫폼 전체 디자인의 프리미엄 감각과 반응형 웹 레이아웃 스페이싱을 1%의 흔들림도 없이 견고하게 철저히 수호하기 위해서입니다.

## 3. 정적 무결성 빌드 및 컴파일 통과
- **의사결정**. 5대 핵심 도메인 이식 수술 직후 `npx tsc --noEmit`을 통한 TypeScript 무결성 실증 및 `npm run build`를 이용한 실제 Turbopack 프로덕션 페이지 빌드를 전부 성공적으로 수료하였습니다.
- **이유**. 소스 코드 변경에 의한 런타임 오류나 빌드 중단을 완전히 방지하고 Vercel 실서버 릴리즈 전 무결함을 사전 입증하기 위함입니다.
