# 📝 비활성 월 통제 및 참여 클래스 노출 복구 컨텍스트 노트

## 1. 아키텍처 의사결정 배경
1. **문제 정의.**
   - 특정 월이 CLOSE(비활성화) 상태일 때 사용자가 수업신청 상세 페이지(`page.tsx`)에서 이전/다음달 이동을 통해 오인입하거나, 토글 및 RESERVE 신청 버튼이 그대로 활성화되어 있는 문제가 있었습니다.
   - 또한, 한국어 로케일 환경에서의 연월 표시 형식이 다소 비직관적이었고, CLOSE 상태를 시각적으로 알릴 수 있는 명확한 배너가 없었습니다.
   - 마지막으로, 월간 패스 신청 팝업에서 현재 선택된 달과 다른 달의 패스(예: 6월 패스)를 신청할 때, 참여 클래스 목록이 빈 리스트(`0 / 0개 선택`)로 노출되는 치명적인 데이터 누락 버그가 존재했습니다.

2. **근본 원인 분석.**
   - 수업 상세 페이지(`page.tsx`)에서는 `group.classPaymentSettings?.openMonths` 정보를 가져와 CLOSE 여부를 검증하는 로직이 빠져 있었습니다.
   - 월 이동 버튼(`<`, `>`)이 하드코딩되어 제공되었기 때문에 CLOSE된 월에서도 자유롭게 넘나들 수 있었습니다.
   - 연월 포맷이 영어/한국어 구분 없이 일률적으로 `May(05), 2026`과 같이 영문 중심으로 노출되었습니다.
   - 클래스 리스트 및 패스 리스트의 토글/RESERVE 버튼에 `disabled` 조건이 누락되어 있었습니다.
   - 팝업 내 참여 클래스 목록(`passClasses`)을 매핑할 때, 이미 현재 선택된 달(5월)로 1차 필터링을 완료한 `classes` 배열을 기준으로 삼았기 때문에, 6월 패스에 매핑되는 6월 클래스들이 5월 `classes` 배열에 존재하지 않아 필터링 결과가 무산되며 빈 목록이 노출되었던 것입니다.

3. **해결 방향 및 설계.**
   - `page.tsx`에 `isRegistrationOpen` CLOSE 상태 판단식을 이식하고, 한국어 환경(`language === 'KR'`)에서 연월이 `5월, 2026`으로 노출되도록 분기했습니다.
   - 상세 페이지 내의 `CLASS INFORMATION` 카드에서 이전/다음 달로 넘어가는 네비게이션 버튼을 제거하여 이탈을 원천 차단했습니다.
   - 비활성화 상태 시 콘텐츠 최상단에 로즈 파스텔톤 경고 배너(`bg-rose-50 border-rose-200 text-rose-600`)를 이식하여 직관적인 안내를 제공했습니다.
   - 모든 종류의 클래스 토글 및 RESERVE 버튼에 `disabled={!isRegistrationOpen}`을 완벽하게 바인딩하고 비활성용 스타일을 입혀 신청 통로를 안전하게 잠궜습니다.
   - 패스 신청 팝업 내의 클래스 필터링 타깃을 기존 `classes`에서 실시간 원본 전체 리스트인 `allGroupClasses`로 교체하여 타 월 패스 신청 시에도 포함된 클래스가 완벽하게 체크박스로 노출되도록 수선했습니다.

## 2. 작업 간 특이사항 및 제한조건
- **디자인 편차 0% 보장 (Zero Design Deviation).**
  - 월 이동 화살표 마크업만 정밀 절개하였고, Stitch 디자인 팔레트 및 HSL 기반의 통일성 높은 스타일을 100% 보장합니다.
- **한국어 종결 규칙 준수.**
  - 모든 설명 및 아티팩트 내 한국어 문장의 종결 기호는 영문 매뉴얼 번역 투인 콜론(`:`)의 사용을 철저히 배제하고, 마침표(`.`)로 명확히 끝맺음합니다.
- **단일 작업 규칙 (Single Task Rule) 및 범위 준수.**
  - 사용자가 지시한 '비활성 월 통제 및 참여 클래스 선택 누락 픽스' 범주 내에서만 수술을 단행했습니다.

## 3. 오픈 월 동적 노출 및 파라미터 유실 버그의 근본 원인과 해결
1. **문제 정의.**
   - 현재 5월인 시점에 5월은 CLOSE 상태이고 6월은 OPEN 상태인데, 대시보드나 수강 신청 페이지에 최초 진입할 때 무조건 `new Date()`를 디폴트 값으로 취해 닫힌 5월이 뜨는 버그가 있었습니다.
   - 대시보드에서 6월 뷰를 탐색하다가 하단의 '수업 신청하기' 버튼을 누르면 상세 수강 신청 오버레이가 기동될 때 탐색 중이던 달 정보가 상속되지 못해 강제로 5월로 리셋되는 현상이 있었습니다.

2. **근본 원인 분석.**
   - 대시보드 컴포넌트(`GroupClassDashboard.tsx`) 및 상세 페이지(`page.tsx`) 양쪽 모두 URL 쿼리 스트링에 `month`가 없을 경우 무조건 `new Date()`를 폴백으로 사용하고 있어, 실제 오늘 날짜에 속한 5월이 기본적으로 우선 선택되었습니다.
   - 대시보드에서 '수업 신청하기' 버튼을 눌러 모달 내비게이션(`openClassFlow('apply')`)을 가동할 때, 현재 뷰포트의 연월 정보(`currentMonthStr`)를 아규먼트로 전달하지 않고 단순히 액션만 트리거하여 URL에 `month` 파라미터가 유실되었던 것입니다.
   - 또한 상세 수강 신청 오버레이(`ClubClassSelectionPage`)가 마운트될 때 URL 쿼리 파라미터 `monthParam` 의 변화를 실시간으로 `currentDate` 상태에 동기화해주는 `useEffect`가 누락되어 있었습니다. 이 때문에 마운트 시점에 파라미터 전달 타이밍 딜레이가 발생하면 디폴트 값인 5월로 픽스되어 꼼짝도 하지 않았습니다.

3. **해결 방향 및 설계.**
   - **오픈 월 스마트 디폴트 복구**: `group.classPaymentSettings?.openMonths`에 등록된 활성화 연월 리스트가 존재할 경우, 오늘 날짜가 그에 포함되어 있지 않다면 오늘 날짜 이후의 가장 빠른 미래 오픈 월을 찾아 디폴트 `currentDate` 상태로 보정하는 똑똑한 동적 디폴트 폴백 알고리즘을 대시보드와 상세 페이지 양쪽에 안전하게 장착합니다.
   - **연월 매개변수 유실 차단**: 대시보드의 `onApplyClick` prop 시그니처에 `monthStr: string`을 추가하고, '수업 신청하기' 및 개별 아이템 카드를 클릭할 때 현재 뷰포트의 `currentMonthStr`를 인자로 태워 호출합니다. 부모 컴포넌트(`GroupHome.tsx`)는 이를 전달받아 모달 내비게이션에 `extraParams: { month: monthStr }`을 정상 바인딩하여 상세 오버레이 기동 시 해당 월이 한 치의 오차도 없이 계승되도록 봉합합니다.
   - **쿼리 파라미터 실시간 동기화**: `ClubClassSelectionPage` 내부에 `monthParam` 변화 감지 `useEffect`를 도입하여, URL에 담긴 오픈 월 정보가 상태 저장소로 즉시 반영되도록 구현해 6월 수강 신청 시 상세 페이지도 정확하게 6월 데이터를 로드하도록 해결합니다.
