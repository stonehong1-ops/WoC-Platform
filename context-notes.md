# 클래스 상세 보강 및 버그 해결 의사결정 노트 (Context Notes)

## 1. 강사 배지 `inst.userId` 바인딩 보강
- **의사결정**. `ClassDetail.tsx` 내 `UserBadge`로 흘러 들어오는 `uid` 속성 값에 `inst.id || inst.uid || inst.userId || ''` 로 `userId` 키 바인딩을 추가로 기입합니다.
- **이유**. Firestore DB에 저장되는 강사의 고유 식별자 키가 어떤 클래스에서는 `id`나 `uid`가 아닌 `userId` 형태로 명명되어 관리되고 있었습니다. 이로 인해 강사 데이터를 캐싱할 때 식별자를 읽어오지 못해 공통 아바타 배지가 미출력되던 결함을 완벽히 보강하기 위함입니다.

## 2. 브라우저/디바이스 popstate 인터셉트 구현
- **의사결정**. `ClassDetail` 모달이 열릴 때 브라우저 히스토리에 가상 상태를 `pushState`로 추가하고, 디바이스 뒤로가기로 발생하는 `popstate` 이벤트를 포착하여 모달 창만 닫히게 통제합니다.
- **이유**. 모바일이나 데스크톱 웹 환경에서 사용자가 클래스 상세 모달을 닫으려고 습관적으로 물리 뒤로가기 키를 누를 때, 상세 팝업만 온전히 닫히는 대신 페이지 전체가 완전히 이전 메뉴나 엉뚱한 페이지로 강제 이탈해 버리는 치명적인 UX 저해 현상을 기술적으로 원천 통제하기 위해서입니다.

## 3. 조건부 렌더링 Falsy Number `0` 출력 버그 박멸
- **의사결정**. `(itemDetail.maxCapacity || itemDetail.leaderCount || itemDetail.followerCount) && (...)` 와 같은 조건부 렌더링 식을 `!!(...) && (...)` 또는 삼항 연산자로 변환하고, `sched.content && ...`를 삼항 연산자로 교체합니다.
- **이유**. Javascript/React의 특징으로 인해 `&&` 평가 시 피연산자의 값이 숫자 `0`인 경우, 전체 조건식이 `0`으로 평가되면서 React가 화면에 문자 그대로 `0`을 인쇄하는 고질적인 렌더링 버그가 유발되었습니다. 데이터베이스 상 정원 혹은 설명 내용이 설정되지 않아 `0`이 반환되는 시점에 시각적 노이즈를 근절하기 위한 무결성 코딩 가이드라인입니다.
