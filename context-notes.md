# WoC 0ms 안착감 극대화 리팩토링 의사결정 노트 (Context Notes)

## 1. 폰트 로드 방식의 일대 전환
- **의사결정**: `GroupHome.tsx` 컴포넌트 하단의 `style jsx global` 하드코딩 `@import`를 제거하고 Next.js 전역 `next/font`로 통합합니다.
- **이유**: 컴포넌트가 마운트될 때마다 외부 구글 폰트 서버로부터 폰트를 매번 무단 다운로드하거나 파싱하는 비효율을 없애고, 브라우저가 화면을 렌더링하기 전 전역 캐시된 폰트를 0ms 만에 텍스트에 적용하게 하여 FOUT(폰트 튕김)을 근절합니다.

## 2. Overlay Stacking Context 영구 상주화 (will-change)
- **의사결정**: `{selectedGroup && ...}` 조건부 마운트 구조를 폐기하고 항상 DOM에 미리 심어둡니다. 그리고 최외각 태그에 `will-change: opacity, transform;`을 마킹합니다.
- **이유**: 브라우저의 Main Thread 락(Long Task)은 DOM 트리가 급속도로 추가/삭제되거나 fixed 레이아웃이 급하게 합성 레이어(Composite Layer)로 분리될 때 심각하게 유발됩니다. 오버레이를 항상 얹어 둠으로써 CPU 부하를 제로화하고, `opacity`만 조절해 GPU 픽셀 셰이더 가속 연산만 타게 함으로써 모바일 기기에서도 프레임 저하 없이 0ms 즉시 노출을 유도합니다.

## 3. 지연 렌더링(Deferred Render) 및 널 세이프(Null-Safety) 장치 탑재
- **의사결정**: `selectedGroup`이 `null`일 때도 오버레이가 사라지지 않고 항상 DOM에 남도록 변경하기 때문에, `activeGroup` 로컬 지연 상태를 선언하여 `group` Props가 `null`이 될 때도 "직전 그룹 데이터"를 닫히기 전까지 그대로 박제시킵니다.
- **이유**: 오버레이가 페이드 아웃되는 180ms 정도의 찰나 시간 동안 화면 내부 텍스트나 레이아웃이 텅 비거나 크래시가 나지 않고 아름다운 시각 무결성을 유지하며 닫히기 위해서 필수불가결한 설계 장치입니다.

## 4. 500ms 지연 프리로드 제거 및 KeepAlive 탭 관리
- **의사결정**: 진입 500ms 뒤 강제적으로 모든 탭의 동적 컴포넌트를 한꺼번에 마운트하던 기존 로직을 제거하고, 탭 터치 시 Lazy Mount 및 `hidden` 속성 KeepAlive 구조로 전면 전환합니다.
- **이유**: 한꺼번에 대규모 컴포넌트 트리가 추가 렌더링될 때 발생하는 병목 현상을 소멸시키고, 사용자가 클릭해 들어간 탭만 신속히 캐시해 두고 다음부터 0ms 순간 로딩을 보장하기 위함입니다.
