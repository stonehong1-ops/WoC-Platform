# 📝 클래스 상세보기 모달 런타임 렌더링 무결성 패치 컨텍스트 노트

## 1. 아키텍처 의사결정 배경
1. **문제 정의.**
   - 클래스 상세보기 모달을 초경량화한 이후, 리얼 배포 환경에서 클래스 카드를 클릭해도 아무런 상세 정보 모달이 나타나지 않거나 화면이 먹통이 되는 런타임 오류가 발생함.
   
2. **근본 원인 분석.**
   - **z-index 레이아웃 중복 충돌.** `ClassDetail` 모달 내부에서 `Portal`을 통하여 화면 최상단 전체를 덮는 `fixed inset-0 z-[9999]` 레이아웃이 완비되어 있으나, 부모 `GroupHome.tsx`에서도 중복으로 `fixed inset-0 z-[9999] bg-background` 래퍼 div로 모달을 감싸서 렌더링하고 있었음. 이로 인해 렌더링 순서와 z-index 누적 맥락상 자식이 부모의 빈 래퍼 뒷면으로 숨어버리거나 표출이 무산됨.
   - **Next.js useSearchParams Suspense 누락.** 넥스트 앱 라우터 환경에서 클라이언트 컴포넌트인 `ClassDetail` 내부 탑레벨에 `useSearchParams` 훅이 호출되고 있으나, 부모 트리에서 이를 차단 및 우회할 `<Suspense>` 경계 바운더리가 없었음. 이 때문에 프로덕션 빌드/하이드레이션 과정에서 일시 정지(Suspend)가 발생하여 클라이언트 사이드 자바스크립트 렌더링 트리가 셧다운되어 클릭 먹통이 됨.
   - **URL 상태 동기화 누수.** 모달을 닫을 때 `classFlow` 파라미터만 제거하고 `modal` 파라미터는 제거하지 않아 다음 모달 렌더링 시점에 쿼리 상태 추적이 모호해지고 꼬임이 생길 수 있었음.

3. **해결 방향.**
   - 부모 `GroupHome.tsx`에서 모달 렌더링 시 z-index를 교란하는 중복 div 레이아웃을 걷어내고 순수한 컴포넌트 마운트 구조로 정상화함.
   - 클라이언트 사이드 하이드레이션 오류를 막기 위해 `<React.Suspense fallback={null}>`을 바인딩함.
   - 상세 모달을 닫을 때 `classFlow`와 `modal` 쿼리를 동시에 안전하게 청소하고 뒤로가기 트랩용 `active=true` 주소를 안전 복원하는 닫기 콜백을 전달함.

## 2. 작업 간 특이사항 및 제한조건
- **디자인 편차 0% 보장 (Zero Design Deviation).**
  - 본 조사는 렌더링 수명 주기와 모달 라우팅 흐름상의 꼬임만 제거하며, 기존에 이식된 수려한 정보창 외형 구조와 Stitch 디자인 시스템 코드는 단 1픽셀도 임의 수정하지 않음.
- **한국어 종결 규칙.**
  - 모든 설명과 텍스트 문맥 끝에 콜론(`:`) 기호 종결을 철저히 배제하고 마침표(`.`)로 종료함.
