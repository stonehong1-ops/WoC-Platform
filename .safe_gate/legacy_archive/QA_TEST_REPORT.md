# WoC 플랫폼 통합 QA 테스트 리포트 및 체크리스트

본 문서는 플랫폼의 모든 페이지, 컴포넌트, 상호작용 요소(버튼, 폼, 모달)에 대한 무결성(Integrity), 디자인 일관성(Zero Deviation), 그리고 데이터 통신(Firebase) 상태를 철저히 검증하기 위해 수립된 **전체 시스템 QA 테스트 마스터플랜**입니다. 

---

## 1. 전역 레이아웃 및 공통 컴포넌트 (Global UI/UX)

### 1.1. 상단 헤더 (`Header.tsx`)
- [ ] **뒤로가기/메뉴 버튼 동작**: 각 라우트별 렌더링 상태 확인 및 클릭 시 Drawer 정상 호출 여부
- [ ] **페이지 타이틀 동기화**: `usePathname`에 따른 `headlineKey` 다국어(`t()`) 매핑 정확성 검증
- [ ] **우측 액션 버튼 정합성**: Location(Map), Language Toggle, Chat(Unread 뱃지), Helpdesk, Search 클릭 시 정확한 라우트 이동 확인
- [ ] **스크롤 상호작용**: 스크롤 다운/업 시 헤더 숨김 및 그림자(`shadow`) 효과 발생 여부 점검 (Zero Deviation 기준)

### 1.2. 하단 내비게이션 바 (`GlobalNavigation.tsx`)
- [ ] **액티브 상태 트래킹**: URL 변경 시 현재 속한 최상위 메뉴(World, Market, Now, Lounge, Groups, My)의 아이콘과 텍스트 하이라이트 전환 확인
- [ ] **서브 메뉴 스크롤**: 상단 서브 메뉴(예: Shop, Resale, Rental, Stay) 활성화 시 터치/마우스 스크롤 정상 작동 확인
- [ ] **알림 배지 정확성**: `todoCount`가 제거된 순수 `unreadCount`가 배지에 실시간(Snapshot)으로 정확히 반영되는지 확인
- [ ] **Safe Area 렌더링**: 모바일 환경(iOS/Android) 하단 홈 바(Safe Area) 영역과의 간섭 여부 점검 (`env(safe-area-inset-bottom)`)

### 1.3. 다국어 (i18n / `LanguageContext.tsx`)
- [ ] **언어 토글 (EN/KR)**: 버튼 클릭 즉시 전역 컨텍스트가 변경되며 UI 깜박임 없이 언어가 치환되는지 검증
- [ ] **하드코딩 텍스트 스캔**: 모든 버튼, 플레이스홀더, 알림창에서 `t()` 함수 누락된 텍스트가 있는지 컴포넌트별 전수 조사

## 2. 모듈별 심층 테스트 체크리스트 (Routing & Features)

### 2.1. World (커뮤니티 코어)
- [ ] `/home`: 홈 피드 데이터 렌더링, 페이지네이션/무한 스크롤 성능 테스트
- [ ] `/plaza`: 게시판 레이아웃 정렬 상태, 작성/수정/삭제 시 낙관적 업데이트 및 Firestore 권한(Rules) 테스트
- [ ] `/venues`: 지도 API 렌더링, 핀 마커 클릭 시 모달 팝업 상태 체크
- [ ] `/people`: 사용자 프로필 리스트 출력, 필터링(지역/역할) 기능 정상 작동 여부

### 2.2. Market (상거래 및 공간 대여)
- [ ] `/shop`: 상품 리스트 그리드 간격(Stitch 시스템), 상품 상세 페이지 라우팅 및 가격 변환기(Currency) 확인
- [ ] `/resale`: 중고거래 이미지 업로드 용량 제한 및 최적화(WebP 변환) 확인
- [ ] `/rental` & `/stay`: 예약 달력 팝업, 시간/날짜 선택 시 상태 충돌 여부 및 예약 확정(Transaction) 처리 검증

### 2.3. Now (액티비티 및 실시간 기능)
- [ ] `/social` & `/events`: 이벤트 캘린더 동기화, 참여 버튼 클릭 시 중복 처리(Debounce) 및 상태 갱신 테스트
- [ ] `/class`: 클래스 신청/취소 프로세스, 정원 초과 시 버튼 비활성화 로직 검증
- [ ] `/live`: 스트리밍/라이브 채팅 기능 마운트 상태, 스트림 종료 후 UI 롤백 테스트

### 2.4. Lounge (사진 및 탐색)
- [ ] `/pics`: 갤러리 뷰 렌더링(Masonry 레이아웃) 깨짐 현상 확인, 이미지 지연 로딩(Lazy Loading) 적용 확인
- [ ] `/lost`: 분실물 신고 폼 필수 값 검증(Validation), 업로드 이미지 용량 초과 에러 핸들링
- [ ] `/hub` & `/explore`: 탐색 메뉴 카테고리 필터링 정확도 확인

### 2.5. Groups (그룹 모듈)
- [ ] `/groups/[id]`: 그룹 상세 페이지 접근 권한(멤버/비멤버/관리자)에 따른 컴포넌트 렌더링 차이 검증
- [ ] **클래스 셋팅 (`GroupClassEditor`)**: 
    - [ ] 관리자 전용 메뉴 진입 권한 체크
    - [ ] Bank Information 폼 값 저장 및 다국어 맵핑 확인
    - [ ] "CLASS" vs "CLASS SETTING" 분리에 따른 탭 인덱스 꼬임 방지 확인
- [ ] **멤버십 가입**: 결제 혹은 가입 버튼 클릭 시 데이터 일관성 유지 (Race Condition 방지)

### 2.6. My (개인화)
- [ ] `/profile`: 사용자 닉네임, 프로필 이미지 변경 시 스토리지 업로드 및 DB 동기화 실시간 확인
- [ ] `/history`: 사용 내역, 신청 내역 리스트 출력 (빈 화면 시 Empty State UI 확인)
- [ ] `/wallet`: 보유 크레딧/티켓 계산식 정합성, 소수점 처리 오류 점검

### 2.7. System (채팅 및 알림)
- [ ] `/notification`: 
    - [ ] 알림 리스트 렌더링 속도 점검
    - [ ] **"모두 읽기" 버튼 클릭 시 `writeBatch` 정상 작동 여부 (Firestore 400개 제한 준수 확인)**
    - [ ] 배지 카운트 0 초기화 실시간 검증
- [ ] `/chat`: 1:1 및 그룹 채팅방 입장 시 스크롤 최하단 이동 확인, 안 읽은 메시지 카운트 로직 검증
- [ ] `/search`: 검색어 입력 시 디바운스(Debounce) 동작, 결과 없음 시 에러 핸들링

## 3. 데이터 흐름 및 예외 처리 (Exception & Firebase)

### 3.1. Firebase Firestore Security Rules
- [ ] 모든 데이터 쓰기(Write/Update/Delete) 동작 시 인가되지 않은 사용자(Unauthenticated)의 요청이 차단되는지 콘솔 에러 확인.
- [ ] 읽기(Read) 시 자신의 데이터만 가져오는지 쿼리 필터링(`where('targetUserId', '==', user.uid)`) 검증.

### 3.2. 상태 충돌 및 로딩 UI
- [ ] 버튼 다중 클릭(따닥) 시 스피너 로딩 UI 렌더링 및 클릭 이벤트 차단(Disable) 확인.
- [ ] 존재하지 않는 라우트 접근 시 404 페이지 렌더링 및 에러 바운더리(`error.tsx`) 전환 정상 작동 확인.

### 3.3. 모바일 기기 호환성 (Responsive Design)
- [ ] 데스크톱 환경(Chrome)과 모바일 뷰(Viewport) 간의 디자인 편차(Zero Deviation) 검증.
- [ ] 모바일 소프트 키보드 활성화 시 인풋 필드가 가려지는 이슈(Scroll to Element) 확인.
