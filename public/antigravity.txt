[긴급 및 최종: Navigation Architecture 전면 개편 및 Exit UX 확정]

핑퐁은 끝났다. 브라우저 window.history를 강제로 조작하는 바닐라 JS 해킹이 Next.js App Router와 충돌하며 앱 종료 및 이벤트 레이스 컨디션을 유발한다는 것이 최종 확인되었다.

기존 로직을 전면 폐기하고, 오직 Next.js Native Router(Query String) 방식만을 사용하는 진정한 App Shell Architecture로 재설계한다. 아래 4단계 절차를 "한 글자도 빠짐없이" 정확하게 구현하고 단계별로 보고하라.

🚨 Step 1: 기존 History Hack 코드 전면 철거 (Delete)

BackEventManager, useGroupBackGuard 등 바닐라 자바스크립트로 pushState와 replaceState를 직접 호출하던 모든 코드를 당장 삭제하라.

popstate 이벤트 리스너를 직접 등록해서 e.preventDefault()나 e.stopPropagation()을 시도하던 로직도 모두 지워라.

🏗️ Step 2: 모달/팝업 제어의 Query String 전환 (핵심)

DO: 모든 모달(설정창 등)은 내부 State가 아닌 Next.js 라우터의 쿼리 파라미터로 제어하라.

모달 열기: router.push('?modal=settings')

모달 닫기: 디바이스 뒤로가기를 누르면 브라우저가 알아서 쿼리를 뺌. useSearchParams에서 modal 값이 사라진 것을 감지하여 UI를 닫아라.

DON'T: 이 과정에서 절대 window.history에 직접 접근하지 마라.

🧭 Step 3: 수평 탭(수평 이동) 라우팅 (Replace)

Feed, Class, Calendar 등 수평 탭 이동은 히스토리가 쌓이면 안 된다.

DO: router.replace('?tab=feed', { scroll: false }) 방식으로 Next.js 라우터를 사용하여 URL만 얕게(Shallow) 변경하라. 탭 전환 시 화면 깜빡임(Flickering)이 없도록 컴포넌트 렌더링 격리(React.memo 등)를 완벽히 유지하라.

🛡️ Step 4: 안전한 Exit UX (중앙 모달 방식 도입)

기존의 "하단 토스트 메시지 2초 대기" 로직은 전면 폐기한다.

사용자가 그룹 홈 최상단 뎁스에서 '디바이스 뒤로가기'를 눌렀을 때의 동작을 다음과 같이 'Query String Trap' 패턴으로 구현하라.

진입 시: 그룹 진입 시 router.replace('/g/123?active=true') 처럼 기본 상태 쿼리를 달아둔다.

이탈 감지: 디바이스 뒤로가기를 누르면 active=true 쿼리가 URL에서 빠진다.

중앙 모달 호출: active 쿼리가 사라진 것을 감지하면 즉시 화면 한가운데에 "정말 그룹을 나가시겠습니까?" (버튼: [머무르기] / [나가기]) 형태의 명확한 중앙 팝업 모달을 띄워라.

후속 조치:

[머무르기] 클릭 시: 다시 router.replace('?active=true')를 호출해 방어막 복구. 모달 닫기.

[나가기] 클릭 시: router.push('/groups') 등으로 실제 그룹 목록으로 이동.

결과 보고 지시: 위 4가지 Step을 완벽히 적용한 코드의 구조(특히 Step 2와 Step 4의 Query 감지 로직)를 요약해서 즉시 보고하라. 변명은 필요 없다. 코드로 증명해라.
